# Стандарт Документаційного Сайту (Docs Site Standard)

> **Призначення:** Цей документ описує стандарт побудови живого сайту документації для будь-якого пакету або додатку в екосистемі `nan.web`. Цей сайт інтегрується з Пісочницею (Sandbox) для інтерактивних прикладів, підтримує багатомовність (i18n) та інтегрований з Економікою Платформи (Theme Store).

---

## 🎯 Філософія

мИ відмовляємося від "документації як статичного тексту". Документація — це **живий артефакт**, який:

1. **Виконується** — `README.md.js` (ProvenDoc) генерує `README.md` з тестованих прикладів.
2. **Інтерактивна** — Кожен приклад має Live Preview у Sandbox.
3. **Багатомовна (i18n)** — Знання має бути доступне кожною мовою (UK, EN за замовчуванням).
4. **Економічна** — Документація є точкою входу в **Theme Store**, де користувач може купити або продати налаштовану тему.
5. **Універсальна** — Дизайн, створений у Sandbox одного пакету, може бути застосований до всієї екосистеми (Universal Themes).
6. **Комерціалізована** — Документація є редактором для тем, які можна продавати як разово, так і за підпискою.
7. **Публікується автоматично** — Збирається як SSG і деплоїться на GitHub Pages / Vercel / Cloudflare.
8. **Перевіряється** — Якщо код змінився, а документація ні — тести падають.

## 📦 Структура документації пакету

Кожен пакет повинен мати таку структуру:

```
packages/{name}/
├── README.md                    # 🔴 Генерований (НЕ редагувати вручну!)
├── src/README.md.js             # 🟢 ProvenDoc Manifest — джерело істини
├── docs/
│   ├── uk/README.md             # Український переклад (LLM-генерований)
│   ├── site/                    # SSG сайт документації
│   │   ├── index.html           # Entry point
│   │   ├── pages/               # Розділи документації
│   │   │   ├── getting-started.md
│   │   │   ├── api.md
│   │   │   ├── examples.md
│   │   │   └── changelog.md
│   │   └── sandbox/             # Інтеграція з Sandbox IDE
│   │       └── index.html       # Вбудований Sandbox з живими прикладами
│   └── {lang}/README.md         # Додаткові переклади
├── .datasets/
│   └── README.dataset.jsonl     # LLM-ready dataset
├── system.md                    # Архітектурні інструкції (укр.)
└── economy.md                   # (Опційно) Економічна модель теми/пакета
```

## 🌐 Багатомовність (i18n)

Документаційний сайт зобов'язаний підтримувати декілька мов:

- **Перемикач мов:** Має бути присутнім у навігаційній панелі.
- **Джерело:** Переклади беруться з `docs/{lang}/` та `@nan0web/i18n` словників.
- **Принцип:** тИ не перекладаєш слова, тИ доводиш, що знання існує у кожній мові.

---

## 💰 Інтеграція з UI Economy (Theme Store)

Документація — це маркетинговий інструмент. Кожен Docs Site має:

1. **Theme Preview:** Можливість миттєво побачити сайт документації у різних темах з маркетплейсу.
2. **Export to Store:** Кнопка в Sandbox, що дозволяє дизайнеру запакувати налаштовані CSS-змінні та виставити їх на продаж.
3. **Subscription Model:** Теми можуть продаватися за моделлю підписки (напр. $5/міс). Це забезпечує підтримку теми при виході нових компонентів або пакетів у екосистемі.
4. **Commission:** Платформа NaN0Web утримує **18%** комісії з продажу тем (82% йде автору).

## 📄 Рівні Документації (від простого до повного)

### Рівень 1: ProvenDoc (Мінімальний — Обов'язковий)

Кожен пакет **зобов'язаний** мати `src/README.md.js`. Це мінімальний рівень, який автоматично:

- Генерує `README.md` із тестованих прикладів.
- Генерує `.datasets/README.dataset.jsonl` для LLM.
- Перекладається в `docs/uk/README.md`.

**Стандарт:** [provendocs.md](./provendocs.md)

### Рівень 2: Docs Site (Розширений — Рекомендований)

Для зрілих пакетів (RRS ≥ 324) створюється повноцінний сайт документації:

| Розділ              | Джерело                              | Опис                                          |
| ------------------- | ------------------------------------ | --------------------------------------------- |
| **Getting Started** | Вручну або з ProvenDoc               | Швидкий старт, встановлення, перший приклад   |
| **API Reference**   | З `types/*.d.ts`                     | Автогенерований список класів, методів, типів |
| **Examples**        | З `src/README.md.js` + `playground/` | Живі приклади з Live Preview                  |
| **Architecture**    | З `system.md` + `project.md`         | Архітектурні рішення, діаграми                |
| **Changelog**       | З `CHANGELOG.md` або git log         | Історія змін                                  |
| **Sandbox**         | Вбудований Sandbox IDE               | Інтерактивна пісочниця компонентів            |

### Рівень 3: Full Portal (Максимальний — для .app додатків)

Для додатків (`*.app`) сайт документації стає повноцінним порталом з:

- Навігацією (ліва бокова панель)
- Пошуком (повнотекстовий)
- Версіонуванням (перемикач між версіями API)
- Інтернаціоналізацією (перемикач мови)
- Змінною темою (light | dark | high-contrast | auto)
- **Store Integration:** Прямий доступ до куплених тем та можливість публікації нових.

## 🛠 Технічний Стек

### SSG Генератор

Документація збирається на базі **`ui-lit`** (найшвидший рендерер в екосистемі). Не використовується жоден зовнішній фреймворк (Next.js, Docusaurus, VitePress тощо) — мИ будуємо власний SSG від `ui-lit`.

```
Вхід: Markdown (.md) + ProvenDoc (.md.js) + Types (.d.ts)
       ↓
Процесор: @nan0web/markdown → AST → ui-lit renderItem()
       ↓
Вихід: Статичні HTML файли (docs/site/)
       ↓
Деплой: GitHub Pages / Vercel / Cloudflare Pages
```

### CSS Стилізація

Документаційний сайт використовує **Base UI** рівень пакету:

- `--co` — акцентний колір бренду (для навігації, посилань)
- `--ba-surface` — фон карток з прикладами
- `--co-text`, `--co-muted` — основний та вторинний текст
- **High Contrast:** Спеціальний режим для покращення доступності, що використовує лише контрастні кольори.
- Підтримка `light | dark | high-contrast | auto` тем.

### Live Code Examples

Приклади з `src/README.md.js` вбудовуються у сайт як інтерактивні блоки:

```html
<nan0-code-example src="README.md.js#example-1">
  <!-- Код прикладу відображається зверху -->
  <!-- Результат виконання — знизу (Live Preview) -->
</nan0-code-example>
```

## 🔗 Інтеграція з Sandbox (Web + CLI)

Sandbox не є окремою сторінкою. Він **вбудовується** безпосередньо в сторінки документації:

1. **Web Sandbox:** Сторінка `Examples` містить Sandbox-фрейм для кожного компонента.
2. **CLI Sandbox (OlmuiInspector CLI):** Для CLI-пакетів документація вбудовує xterm.js термінал з запущеною пісочницею `play/sandbox.js`. Це дозволяє протестувати CLI досвід прямо в браузері.
3. Кожен компонент на сторінці API має кнопку "Open in Sandbox", яка перемикає на повноекранний редактор з Live Preview.
4. Варіації (`SandboxVariant`), створені користувачем, зберігаються локально і доступні після перезавантаження, а також можуть експортуватись і імпортуватись у свій формат.
