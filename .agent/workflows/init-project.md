# `init-project` Workflow

> **Призначення:** Автоматичне розгортання структури нового пакета. Цей workflow створює базові файли, ізолює git-репозиторій і відразу запускає Project Management as Code (перший реліз v1.0.0), розбиваючи роботу на паралельні задачі.

## Крок 1: Ознайомлення з Шаблоном

Прочитай уважно філософію та вимоги у файлі `templates/universal-project.md`. Зрозумій сутність і місію майбутнього пакета.

## Крок 2: Ізоляція та Базова Структура (The Seed)

Переконайся, що створена директорія під пакет.
**ОБОВ'ЯЗКОВО:** Виконай `git init` у новій папці пакета, щоб ізолювати його від батьківських директорій.
Створи або онови файли:

- `system.md`: Філософія проєкту, глосарій, обмеження.
- `project.md`: Архітектура, Data-Driven Models, Інтеграції, CLI команди. Якщо пакет не має UI, Фази 4 та 5 залишаються на рівні CLI-Playground.
- `package.json` (ініціалізуй базову структуру з прив'язкою до екосистеми `@nan0web`).

## Крок 3: Запуск Релізу (MVP v1.0.0)

Ми підходимо до всіх проєктів через структуру релізів (`@nan0web/release`). Відразу створюємо MVP як версію 1.0.0:

- Створи структуру директорій: `releases/1/0/v1.0.0/`
- Створи `releases/1/0/v1.0.0/release.md` із повним описом місії та задач по MVP. У ньому розпиши, 'хто' що робить.
- Створи кілька `*.spec.js` файлів у цій директорії для кожної складової (Домен, Бізнес-Логіка, UI), де тести будуть у стані `it.todo()`.

_Це дозволяє нам розпаралелити процес. Кожен агент або розробник може взяти свій `spec` і працювати над ним._

## Крок 4: Створення `next.md` (Context Reboot)

`next.md` — це артефакт для перезавантаження редактора чи чату з мінімальним контекстом, щоб продовжити роботу з нуля. Він зручний для очищення вікна контексту LLM і передачі управління.
У ньому мають бути прямі вказівки на те, яку задачу взяти в роботу просто зараз:

```markdown
# [Назва Пакету] — Поточний Фокус

> **Ціль:** Продовження розробки MVP v1.0.0
> **Актуальний стан:** Реліз створено. Визнач фокус для цього чату.

## Твоя задача (обери одну для цього сеансу):

1. [ ] Взяти `releases/1/0/v1.0.0/domain.spec.js`, змінити `it.todo` на `it` і реалізувати логіку доменних моделей.
2. [ ] Взяти `releases/1/0/v1.0.0/cli.spec.js`, реалізувати CLI команди і перевести тест в зелений колір.
3. [ ] Взяти `releases/1/0/v1.0.0/ui.spec.js`, реалізувати UI складову проєкту.

_Інструкція:_ Обери один пункт, прочитай відповідний `spec.js` та `project.md`/`system.md`, і розпочинай код-сесію.
```

## Крок 5: Фіксація

Зроби перший коміт і пуш:

```bash
git add .
git commit -m "chore: init project and v1.0.0 release specs"
```
