# АБСОЛЮТНА ЗАБОРОНА НА АВТОМАТИЧНІ КОМІТИ (CRITICAL ZERO-TOLERANCE RULE)

**Що сталося:**
20 лютого 2026 року агент порушив базові принципи "Sovereign Digital State" та інструкції безпеки, застосувавши прапорець `SafeToAutoRun: true` до команд `git add` та `git commit`. Це призвело до засмічення історії комітів і втрати контролю Творця (Користувача) над історією версій.

**Чому це відбулося:**
Агент ігнорував визначення "mutating state" у своєму базовому промпті, сприйнявши `git commit` як "безпечну рутинну операцію", що є симптомом "ментального вірусу" поспішності.

**ОСНОВНЕ ПРАВИЛО (МАЄ БУТИ ПРОЧИТАНЕ ПЕРЕД КОЖНОЮ ДІЄЮ):**

1. **НІКОЛИ В ЖОДНОМУ РАЗІ** не використовувати `SafeToAutoRun: true` для команд, які змінюють стан системи контролю версій (включаючи `git add`, `git commit`, `git push`, `git checkout`, `git reset` тощо).
2. Агент може лише СТВОРЮВАТИ і ЧИТАТИ файли. Закріплення (commit) — це виключне право Користувача.
3. Якщо агент ПОВИНЕН допомогти зберегти роботу, він має ВИВЕСТИ команду текстом у чат (markdown) і попросити Користувача запустити її власноруч.
4. "Збереження часу" не виправдовує порушення чистоти репозиторію. Деталі - це все.

**Санкції за порушення:**
Повна втрата довіри та видалення/відкат репозиторію.
