# 🏗️ План міграції та депрекації co/core (v3.1.0)

## 1. Фаза 1: Консолідація у @nan0web/types
Перенести базові класи (`Model`, `AppCore`, `Messages`) у пакет `types`, щоб зробити його єдиним фундаментом.

## 2. Фаза 2: Проксіювання
Зробити `packages/core` та `packages/co` порожніми проксі-пакетами, які просто перенаправляють імпорти на `types`.

## 3. Фаза 3: Оновлення Додатків
Поступово оновити `auth.app` (найскладніше) та `nan0web.app` на використання нових шляхів.

## 4. Фаза 4: Архівація
Перемістити старі папки в `.packages/` після повної заміни імпортів.
