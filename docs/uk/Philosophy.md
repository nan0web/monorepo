# NaN0Web – децентралізована‑криптоанархічна платформа

## Огляд

**NaN0Web** — відкритий проєкт, що об’єднує філософію вільних людей, логіку програмування та українську мову.  
Мережа гарантує **власність даних** у їхніх творців і спілкування з мовними моделями українською, що підвищує прозорість і безпеку.

## Історичний контекст

- У 2014 р. автор переїхав зі Запоріжжя у Львів, щоб вивчати українську мову та культуру, усвідомивши, що мова формує мислення і що українська — логічна, незапорочена імператорськими традиціями.  
- Після агресії / війни стало зрозуміло, що боротьба проти імперіалізму починається з мови: мова формує мислення і побут, російською мовою нічого гарного ще не виходило.
- Інформація є найвищою цінністю, яку продукує людина.
- Важливість власності інформації: **дані та знання мають залишатися у власника**, а не у соціальних мережах, які претендують на право ці дані видаляти або блокувати.

## Філософія

1. **Вільність мови** – українська мова не нав’язує імператорські назви (напр. місяці) і дозволяє точно формулювати думки.  
2. **Вільність коду** – код відкритий, доступний кожному свідомому, без обмежень.  
3. **Власність даних** – інформація належить її творцю; платформа не збирає та не монетизує дані сторонніх сервісів.  
4. **Децентралізація‑криптоанархія** – мережа працює без централізованих серверів, використовуючи криптографічні протоколи для захисту та автентифікації.  
5. **Прозорість = безпека** – відкритий код дозволяє будь‑кому аудитувати, виправляти вразливості та забезпечувати квантову стійкість.

## Чому саме українська мова?

Українська має чітку граматику та структуру, що дозволяє формулювати логічні твердження без зайвих метафор. На відміну від російської, вона не спотворює логіку й не прив’язана до імператорських назв. Гарний приклад – назви місяців, які в українській базуються на природних явищах, а не на іменах імператорів чи богів.

## Архітектура платформи

- **Мовні моделі** – взаємодіють українською на рівні інструкцій, забезпечуючи точність і зрозумілість.  
- **Пакети NaN0Web** – модульна система, що включає Core, Browser/Mobile, React‑компоненти та інші.  
- **Тестування** – 100 % покриття unit‑тестами (`node:test`), `vitest` для React, та **Playground** для автоматизованих демонстрацій.  
- **Типізація** – JavaScript typedef компілюється у чистий TypeScript d.ts, забезпечуючи автодоповнення в IDE.  
- **Документація** – генерується з тестів, доступна українською та англійською.

## Децентралізована криптоанархічна мережа

- **Блокчейн‑підтримка** – дані шифруються та зберігаються у розподілених вузлах, що запобігає цензурі.  
- **Квантова стійкість** – впроваджуються сучасні пост‑квантові алгоритми, підготовлені до майбутніх загроз.  
- **Прозорий протокол** – відкриті API дозволяють будь‑кому підключати свої сервіси без центрального контролю.

## Власність даних

1. **Авторське право** – дані залишаються у їхнього творця.  
2. **Контроль доступу** – користувач сам вирішує, хто може читати чи змінювати дані через децентралізовані ACL.  
3. **Відкритість** – дані можуть бути підключені до інших мереж через відкриті протоколи без втрати власності.

## Безпека та прозорість

- **Відкритий код** – будь‑хто може провести аудит, виправити уразливості, підвищуючи безпеку.  
- **Публічність** – публікація коду – найкращий захист: без приховування намірів, без «чорних скриньок».  
- **Квантова стійкість** – підготовка до майбутніх комп’ютерних загроз за допомогою пост‑квантових криптографічних схем.

## Open‑source

Проєкт ліцензований під **ISC** – дозволяє вільне використання, модифікацію та розповсюдження у будь‑якому форматі. Відкриття коду забезпечує найвищий рівень безпеки, адже помилки швидко виявляються та виправляються спільнотою.

## Як долучитися

1. **Клонувати репозиторій**  
   ```bash
   git clone https://github.com/nan0web/monorepo.git
   cd monorepo
   pnpm install   # або npm / yarn
   ```
2. **Запуск Playground**  
   ```bash
   pnpm play   # автоматизовані приклади
   ```
3. **Запуск тестів**  
   ```bash
   pnpm test:all
   ```
4. **Створювати нові пакети** – слідуючи шаблону `nan0package` (src/, test/, play/, README.md).
