# Підготовка середовища та розгортання Knowledge Base (MCP)

Цей документ описує процес передачі архітектури `nan0web` іншому розробнику. Він охоплює етапи клонування, встановлення залежностей та налаштування локальної AI-екосистеми з використанням технології Model Context Protocol (MCP) та локальної LLM (LM Studio / Ollama).

## Крок 1: Передача платформи (Клонування)

Оскільки проект має архітектуру монорепозиторію, вам не потрібно клонувати окремі пакети. Необхідно завантажити весь корінь платформи.

1. Зробіть `git clone` (або Fork) всього репозиторію на локальну машину.
2. Перейдіть до кореня монорепозиторію і встановіть залежності для всіх пакетів:
```bash
npm install 
# або pnpm install, якщо використовується pnpm
```

## Крок 2: Підключення локальної LLM (EMBEDDER)

Для роботи пошуку по знаннях (векторна база даних) платформі потрібен доступ до моделі, яка здатна робити Embeddings (перетворювати текст на вектори).

1. Запустіть **LM Studio** або **Ollama** локально.
2. Активуйте локальний сервер (Local API Server). Зазвичай це:
   - LM Studio: `http://localhost:1234/v1`
   - Ollama: `http://localhost:11434/v1`

### Збереження конфігурації (EMBEDDER_URL)
Щоб не прописувати адресу сервера щоразу, ви можете зберегти конфігурацію:
- **Глобально (на рівні користувача):** Додайте `export EMBEDDER_URL="http://localhost:1234/v1"` у ваш `~/.zshrc` або `~/.bash_profile`.
- **Локально (на рівні проекту):** Створіть файл `.env` у корені монорепозиторію з рядком:
```env
EMBEDDER_URL=http://localhost:1234/v1
```

## Крок 3: Встановлення MCP Server (Інтеграція з IDE)

Платформа має готовий скрипт для автоматичного підключення інструментів AI-пошуку до вашої IDE (Cursor, Windsurf, Claude Desktop, Antigravity).

Перейдіть до пакету `ai` та запустіть встановлення:
```bash
cd packages/ai
npm run mcp:install
```

> **Важливо:** Цей скрипт автоматично запише `EMBEDDER_URL` та шляхи до сервера безпосередньо в конфігураційні файли ваших IDE. Таким чином, конфігурація зберігається "глобально" для вашого редактора коду, і вам не доведеться її налаштовувати повторно. Після встановлення **обов'язково перезапустіть IDE**.

## Крок 4: Індексація Платформи

Щоб LLM могла миттєво знаходити будь-який код чи архітектурний документ, потрібно проіндексувати всі `project.md`, вихідні файли (`source`) та документацію.

Знаходячись у папці `packages/ai`, виконайте:
```bash
npm run index
# Або з кореня: npx nan0ai index
```

Команда перевірить кеш і згенерує векторні сліди для всіх пакетів у монорепозиторії.

## Крок 5: Перевірка та Валідація (Як переконатися, що все працює)

Після того, як ви виконали всі кроки, ви можете легко перевірити цілісність системи трьома способами:

### Спосіб А: Через CLI-асистента (`nan0ai`)
Платформа має вбудований термінальний інтерфейс для взаємодії з базою знань.
Відкрийте термінал у корені проекту і спробуйте наступні команди:

- **Перевірка наявності метаданих:**
  ```bash
  npx nan0ai show
  ```
- **Список проіндексованих файлів:**
  ```bash
  npx nan0ai ls
  ```
- **Локальний пошук через термінал:**
  Здійсніть прямий запит до векторної бази, щоб перевірити якість пошуку (схоже на веб-пошук, але локально):
  ```bash
  npx nan0ai search "Що таке OLMUI?"
  ```

### Спосіб Б: Валідація всередині IDE (MCP)
Якщо ви правильно встановили MCP (Крок 3), ваша IDE тепер має новий інструмент.
1. Відкрийте чат з AI у вашій IDE (наприклад, Windsurf або Cursor).
2. Перевірте, чи доступні інструменти `mcp_nan0web-knowledge_search_knowledge_base` та `mcp_nan0web-knowledge_get_resource`.
3. Задайте питання: _"Знайди у базі знань nan0web інформацію про архітектуру додатків і поясни її"_.
4. Переконайтеся, що LLM автоматично викликала локальний MCP-сервер, прочитала документи з бази і надала відповідь, опираючись на ваші локальні файли.

### Спосіб В: Перевірка через Тимчасову Директорію
Якщо ви готуєте проект до передачі іншій людині і хочете бути впевнені:
1. Скопіюйте папку `nan.web` в `/tmp` (не копіюйте папку `node_modules`).
2. Виконайте там `npm install`.
3. Виконайте індексацію `npx nan0ai index`.
4. Якщо не виникло помилок збирання чи індексації — проект повністю ізольований і готовий до передачі "під ключ".
