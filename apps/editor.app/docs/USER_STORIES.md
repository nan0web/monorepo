# User Stories: NaN0 Editor

## 1. Рекурсивне редагування (Stack of Modals)

**Як** користувач-розробник,  
**Я хочу** мати можливість відкрити вкладений об'єкт у новому вікні, не втрачаючи контексту батьківського документа,  
**Щоб** швидко редагувати зв'язані дані.

- [ ] **Story 1.1**: Відкриття першої модалки.
- [ ] **Story 1.2**: Відкриття вкладеної модалки (Level 2+).
- [ ] **Story 1.3**: Закриття верхнього шару з поверненням даних.
- [ ] **Story 1.4**: Перевищення ліміту глибини (7 рівнів).

## 2. Model-Driven Form Generation

**Як** контент-менеджер,  
**Я хочу**, щоб форма редагування підлаштовувалася під схему моделі,  
**Щоб** мені не потрібно було вручну налаштовувати типи полів.

- [ ] **Story 2.1**: Автоматичний рендеринг `text`, `string`, `list`.
- [ ] **Story 2.2**: Відображення `help` текстів під полями.

## 3. Persistence & Strategies

**Як** адміністратор,  
**Я хочу** обирати стратегію збереження (Cache, Commit, Git),  
**Щоб** контролювати процес публікації змін.

- [ ] **Story 3.1**: Збереження в локальний кеш (IndexedDB).
- [ ] **Story 3.2**: Атомний коміт у базу даних.
