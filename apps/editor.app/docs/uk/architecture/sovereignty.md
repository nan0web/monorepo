# 🔒 Суверенність (Sovereignty)

> Повний контроль над даними незалежно від платформи, хостингу чи провайдера.

---

## Визначення

Суверенність означає, що користувач **завжди** є єдиним власником своїх даних. Дані не залежать від хмарного сервісу, не заблоковані у пропрієтарному форматі і можуть бути перенесені між середовищами без втрат.

## Як це працює в NaN•Web

### 1. DSN (Data Source Name)
Системний маніфест `nan0web.nan0` містить єдину точку входу до даних:

```yaml
system:
  dsn: "fs:./data"
```

DSN визначає **адаптер** (файлова система, IndexedDB, S3, IPFS) та **шлях**. Зміна адаптера не впливає на логіку додатка.

### 2. DB-FS (Database as File System)
Всі дані зберігаються у відкритих форматах:
- **`.nan0`** — YAML-підмножина для конфігурації.
- **`.yaml`** — структуровані дані.
- **`.md`** — текстовий контент (Markdown).
- **`.jsonl`** — потокові дані та датасети.

### 3. Міграція та Портабельність
Оскільки дані — це звичайні файли, їх можна:
- Копіювати між машинами (`rsync`, `scp`).
- Версіонувати через Git.
- Зберігати на зашифрованих носіях.
- Публікувати на IPFS або інших децентралізованих мережах.

## Приклади

### Локальне розгортання (Self-Hosted)
```yaml
system:
  dsn: "fs:./data"
```
Дані зберігаються на диску. Жоден зовнішній сервіс не потрібен.

### Браузерне розгортання
```yaml
system:
  dsn: "indexeddb:editor-app"
```
Дані зберігаються в IndexedDB браузера. Працює офлайн.

### Децентралізоване розгортання
```yaml
system:
  dsn: "ipfs:QmXyz..."
```
Дані доступні через IPFS. Незнищенність та цензуростійкість.

## Аналоги в індустрії

| Платформа | Рівень суверенності | Обмеження |
| :--- | :--- | :--- |
| **Google Docs** | Низький | Дані на серверах Google, vendor lock-in |
| **Notion** | Середній | Експорт можливий, але неповний |
| **Obsidian** | Високий | Локальні файли, але без DB абстракції |
| **NaN•Web** | Максимальний | DSN-абстракція, портабельні формати |
