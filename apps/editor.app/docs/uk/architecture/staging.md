# 📦 Стейджинг (Staging)

> Локальний перегляд та перевірка змін перед публікацією.

---

## Визначення

Стейджинг — це буферна зона між поточною роботою користувача та опублікованим станом бази даних. Зміни зберігаються в окремому просторі (`_staged/`), де їх можна переглядати, редагувати та видаляти без впливу на основну базу.

## Архітектура

```
┌─────────────────────────────────────────┐
│              User Session                │
│                                          │
│  edit() ──► stageDb["_staged/doc.md"]   │
│                    │                     │
│              review & verify             │
│                    │                     │
│  commitChanges() ──► mainDb["doc.md"]   │
│                                          │
│  discard() ──► delete _staged/doc.md    │
└─────────────────────────────────────────┘
```

### Два рівні бази

1. **`stageDb`** — Локальне сховище. У браузері це IndexedDB. На диску — тека `_staged/`.
2. **`mainDb`** — Основна база (файлова система, хмара, IPFS).

Обидві бази реалізують один і той самий інтерфейс `DB`, тому код не залежить від конкретної реалізації.

## Як це працює

### 1. Додавання змін до стейджу

```js
const editor = new EditorModel({}, { db: fs })

// Зміна зберігається в _staged/docs/hello.md
await editor.stageChange('docs/hello.md', '# Оновлений контент')
```

### 2. Завантаження документа (Stage-First)

```js
// Спочатку шукаємо в стейджі, потім в основній базі
const doc = await editor.loadDocument('docs/hello.md')
```

Логіка `loadDocument`:
1. Шукає `_staged/docs/hello.md` у `stageDb`.
2. Якщо знайдено — повертає незбережену версію.
3. Якщо ні — завантажує з `mainDb`.

### 3. Фіксація (Commit)

```js
const result = await editor.commitChanges('Оновив документацію')
// Всі файли з _staged/ переносяться в mainDb
// Стейдж очищується
```

### 4. Відкат (Discard)

```js
// Просто видаляємо файл зі стейджу
await editor.stageDb.deleteDocument('_staged/docs/hello.md')
```

## Візуалізація стану

У UI-адаптерах стейджовані файли відображаються зі спеціальними маркерами:

```
📂 docs/
  📝 hello.md (staged ●)     ← змінений, не опублікований
  📄 README.md                ← чистий, з основної бази
  📝 new-file.md (new ●)     ← створений, ще не існує в основній базі
```
