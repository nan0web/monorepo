# План підготовки @nan0web/editor-app до публікації на GitHub

Цей план описує кроки для приведення структури релізів у відповідність до стандартів NaN•Web та підготовки пакету до відкритої публікації.

## 1. Реструктуризація Релізів (Sharding)
Поточна структура має `releases/v1.0.0`, що порушує стандарт `releases/{major}/{minor}/vX.Y.Z/`.

- [ ] Перемістити вміст `releases/v1.0.0` у `releases/1/0/v1.0.0/`.
- [ ] Перейменувати `tasks.md` на `task.md` (сингулярна форма згідно стандарту).
- [ ] Перейменувати `release.md` на `changelog.md` (якщо це історія змін) або інтегрувати в `task.md`.
- [ ] Перевірити `releases/1/0/v1.0.1/` на відповідність.

## 2. Гігієна та Документація (GitHub Ready)
Відсутні базові файли для публічного репозиторію.

- [ ] Створити `README.md` з описом архітектури (на основі `project.md`).
- [ ] Створити `LICENSE` (ISC License, згідно з іншими пакетами NaN•Web).
- [ ] Оновити `package.json`: додати `repository`, `bugs`, `homepage`, `keywords`.
- [ ] Перевірити `.npmignore` (та `.gitignore`), щоб не допустити витоку `.agent/`, `docs/` (якщо вони внутрішні), тощо.

## 3. Верифікація Стабільності
- [ ] Запустити `npm run test:all`.
- [ ] Перевірити build (`npm run build`).
- [ ] Перевірити CLI (`nan0-editor --help`).

## 4. Підготовка до GitHub (Git State)
- [ ] Перевірити наявність невідстежуваних файлів, які мають бути видалені.
- [ ] Сформувати фінальний звіт для Архітектора.
