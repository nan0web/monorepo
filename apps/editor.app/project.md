# 🏗️ Проєкт: NaN•Web Editor (editor.app)

> **Призначення:** Універсальний інструмент візуалізації та маніпуляції графами нан•даних з підтримкою рекурсивної вкладеності та архітектурою "Database-as-Truth".

---

## 🧭 Фаза 1: Архітектурний Матричний Аналіз

Для створення лідера ринку ми орієнтуємось на ключові можливості 12+ топових систем (див. детальний аналіз у [docs/uk/architecture/index.md](docs/uk/architecture/index.md)):

| Система | Ключова можливість | Еквівалент NaN•Web |
| :--- | :--- | :--- |
| **VS Code** | API Розширень | Поліморфні Дії (`ExplorerAction`) |
| **Figma** | Багатокористувацький Граф | Рекурсивний `ModalStack` |
| **WordPress** | Життєвий цикл Документа | Успадкування через `_.nan0` |
| **Odoo / ERP** | Модель-як-Схема | `EditorModel` + `ModelAsApp` |
| **Antigravity** | Агентна Логіка | Інтегрований `SpecRunner` |
| **GitHub** | Версіонування та Стейджинг | `Local Stage` (stageDb) |
| **Notion** | Блочна структура | Зв'язані Документи |

---

## 📐 Фаза 2: Уніфікована Модель Документа

Ми впроваджуємо **Поліморфний Документ** ([Document.js](src/domain/Document.js)):
- **Мета-дані**: Версіонування, права доступу та успадкування налаштувань директорії (`_.nan0`).
- **Логіка**: Само-валідовані властивості через `Model-as-Schema`.
- **Презентація**: Відв'язані UI-адаптери (`lit`, `cli`), що рендерять одну й ту саму модель залежно від контексту.

---

## 🧭 Фаза 4: Презентаційний сайт (Self-Hosting Demo)

### 1. Мета
Демонстрація "Self-Hosting": використання редактора для редагування власного презентаційного сайту.

### 2. Архітектура "Document-as-Slide"
- Кожен "слайд" — це об'єкт класу `Document`.
- `PresentationModel` агрегує ці документи для візуалізації.
- Редагування контенту слайдів відбувається через стандартний інтерфейс `EditorModel`.
