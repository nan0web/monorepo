## Потік повідомлень (Message Flow - Legacy)

Раніше комунікація UI будувалася навколо серіалізованих повідомлень:

- **`UiMessage`** – абстрактний базовий клас повідомлення.
- **`OutputMessage`** – вихідні дані системи (контент, помилка, пріоритет).
- **`InputMessage`** – вхідні дані користувача.

### Приклад застарілого використання

```js
import { InputMessage, OutputMessage } from '@nan0web/ui'
const input = UiMessage.from({ body: 'Привіт, Користувачу' })
const output = OutputMessage.from({ content: ['Вітаємо у @nan0web/ui'] })
console.info(input) // ← Message { body: "Привіт, Користувачу", head: {}, id: "....", type: "" }
```

### Чому це застаріло?
У новій архітектурі ми використовуємо **Генератори та Інтенти** (`yield ask()`, `yield show()`). Це дозволяє декларативно та агностично до платформи керувати логікою додатку, не маніпулюючи об'єктами повідомлень вручну.
