# Legacy Architecture (v1.0 - v1.11)

> **DEPRECATED CONTENT / ЗАСТАРІЛИЙ КОНТЕНТ**

## Message Flow (Legacy)

UI communication was previously built around serializable messages:

- **`UiMessage`** – abstract message base class
- **`OutputMessage`** – system output (content, error, priority)
- **`InputMessage`** - user input data

### Example of Legacy Usage

```js
import { InputMessage, OutputMessage } from '@nan0web/ui'
const input = UiMessage.from({ body: 'Hello User' })
const output = OutputMessage.from({ content: ['Welcome to @nan0web/ui'] })
console.info(input) // ← Message { body: "Hello User", head: {}, id: "....", type: "" }
```

### Why it is Legacy?
In the new architecture, we use **Generators and Intents** (`yield ask()`, `yield show()`). This allows for a more declarative and platform-agnostic way of handling application logic without manually managing message objects.
