# 🧪 Ворклоу: Isolated Sandbox Verify (ISV)

Цей ворклоу використовується для швидкої відладки та візуальної перевірки окремих UI-елементів без запуску всього додатку.

## 1. Підготовка

- **ЗАБОРОНЕНО** тестувати дрібні зміни одразу у головному `main.js`.
- **ОБОВ'ЯЗКОВО**: Створити або оновити файл у `play/ComponentName.play.js`.

## 2. Сценарій мокання

- Створіть мінімальний `mockDb` або `mockModel`.
- Конфігуруйте екстремальні випадки (дуже довгий текст, 1000+ елементів, порожні дані).

```javascript
// play/Component.play.js
import { html, render } from 'lit'
import '../src/Component.js'

render(html`<my-component .data=${mock}></my-component>`, container)
```

## 3. Запуск

- Виконайте `npm run play`.
- Відкрийте `/play/` у браузері.
- Виберіть потрібний компонент у меню контролів.

## 4. Верифікація

- Перевірте фокус (Tab navigation).
- Перевірте адаптивність (Mobile vs Desktop).
- Створіть снепшот: `npm run gallery`.
- Якщо інтерфейс "пливе" або "вилітає", виправте CSS перш ніж переносити код у `src/`.
