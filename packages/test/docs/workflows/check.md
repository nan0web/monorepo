// turbo-all

1. **Аналіз контексту**: Перевір останні змінені файли. Якщо це компонент або модель, знайди відповідний тест у теці `src/**/*.{test|spec}.{js|py}` або аналогічно у `tests/` і запусти його індивідуально (наприклад, `npx playwright test tests/specific.spec.js` або `node --test --test-timeout=3333 src/model.test.js` або `npx vitest --run src/components/App.test.jsx`). Використовуй таймаути для прискорення процесу, можеш їх визначати практично, починаючи з 3333мс і збільшуючи або зменшуючи. Я вважаю таймаути більше за 9999мс поза межею адекватності у TDD.

2. **Автоматичне виправлення**: Запусти `pnpm fix:all` для приведення проєкту до стандартів DSL проєкту.

3. **Глобальне тестування**: Запусти повний набір тестів командою `pnpm test:all`.

4. **Валідація білду**: Запусти `pnpm build`, якщо build не запускався у test:all, щоб переконатися, що проєкт готовий до релізу.

5. **AGRP Контракти**: Якщо в проєкті є `releases/` з `.test.js` контрактами, запусти `npm run release:verify` для перевірки регресії. Якщо контракти падають — це БЛОКЕР, виправляй в першу чергу.
