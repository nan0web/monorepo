# Playground Inspector Protocol

Цей workflow встановлює глобальний архітектурний стандарт для всіх пакетів (`ui-react`, `ui-lit`, `ui-cli` тощо) щодо перевірки консистентності їхньої пісочниці (Playground/Gallery). 

Головна мета — гарантувати, що код в прикладах (JSX / JS / YAML) на 100% відповідає тому, що реально рендериться на екрані.

Оскільки кожен пакет має власні технології (DOM, Термінал, Canvas), **конкретна реалізація інспектора є локальною відповідальністю пакету**, а цей воркфлоу просто виступає оркестратором.

## Крок 1: Верифікація Контракту (package.json)
Кожен UI пакет повинен мати скрипт `inspect:playground` в `package.json`. 
Перевірте наявність команди:
\`\`\`bash
cat package.json | grep inspect:playground
\`\`\`

Якщо команди немає, допоможіть розробнику її створити:
1. З'ясуйте тип пакету (React, Lit, Terminal).
2. Напишіть скрипт для цього конкретного середовища (напр., Playwright для React/Lit, або Child Process Capture для CLI).
3. Додайте \`"inspect:playground": "node scripts/inspect-playground.js"\` в \`package.json\`.

## Крок 2: Запуск перевірки
Запустіть локальний інспектор, який є специфічним для цього пакету:
// turbo
\`\`\`bash
npm run inspect:playground
\`\`\`

**Очікувана поведінка локального скрипта:**
- Він має проаналізувати всі компоненти в Playground.
- Для КОЖНОГО блоку він перевіряє (1) Візуальний Рендер, (2) Код в прикладі (JSX/JS) та (3) Метадані (YAML).
- Виводить у термінал зручний, покроковий звіт (OK / MISMATCH).

## Крок 3: Аналіз та Виправлення (Zero-Hallucination)
Проаналізуйте вивід з Кроку 2.
- Якщо є помилки \`MISMATCH\`, знайдіть файли з прикладами у даному пакеті (зазвичай \`play/*.jsx\` або файл компонентів).
- **CRITICAL:** Виправте хардкод! Переконайтеся, що дані для рендеру і для тексту-коду беруться з єдиного джерела істини або синхронізовані.
- Переконайтеся, що не зникли необхідні властивості (наприклад, не загублено контекст).

## Крок 4: Фінальна Верифікація
Перезапустіть \`npm run inspect:playground\` після виправлень. Якщо вивід повністю "зелений" (ALL CONSISTENT) – виконання успішно завершено!
