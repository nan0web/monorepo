# Voice Auth Protocol (Короткі голосові запити)

Коли агент має намір використати інструменти для запису або читання файлів поза межами поточного робочого простору (workspace) або коли потребує підтвердження/допомоги від клієнта, він **ПОВИНЕН** завжди ініціювати голосове повідомлення за допомогою інструменту `run_command` з флагом `SafeToAutoRun: true`.

⚠️ **НАЙГОЛОВНІШЕ ПРАВИЛО:** Повідомлення має бути **дуже коротким** (1-2 коротких речення). Жодних довгих абзаців. Користувач сам прочитає конкретні деталі на екрані, йому потрібно лише прийти на звук.

**Мовна Адаптація (Language Agnostic):**
Агент повинен автоматично адаптувати мову повідомлення та голос залежно від мови спілкування з користувачем у поточному чаті.

**Якщо спілкування ведеться Українською:**
Використовуй голос `Lesya (Enhanced)`.

```bash
say -v "Lesya (Enhanced)" "Потрібна допомога."
```
або:
```bash
say -v "Lesya (Enhanced)" "Очікую підтвердження команди."
```

**Якщо спілкування ведеться Англійською (English):**
Використовуй приємний стандартний голос, наприклад `Samantha` або `Alex`.

```bash
say -v "Samantha" "I need your help."
```
або:
```bash
say -v "Samantha" "Waiting for your confirmation."
```

Цей звуковий сигнал є обов'язковим Voice Authorization Protocol (Голосовий Протокол Авторизації). Він служить лише для привернення уваги Архітектора.
