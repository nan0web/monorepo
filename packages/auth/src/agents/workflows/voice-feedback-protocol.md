# Voice Feedback Protocol (Короткі звіти прогресу)

Коли агент виконує комплексні або довготривалі завдання (наприклад, виконання багатоскладових Workflow), він **ПОВИНЕН** періодично повідомляти Архітектору про свій прогрес голосом. 

⚠️ **НАЙГОЛОВНІШЕ ПРАВИЛО:** Повідомлення мають бути **надзвичайно короткими** (1-2 коротких речення максимум). Жодних довгих абзаців чи переліків.

**Мовна Адаптація (Language Agnostic):**
Розпізнавай мову спілкування з користувачем і адаптуй голос та текст звіту:
- Для **Української** використовуй `Lesya (Enhanced)`.
- Для **Англійської** використовуй `Samantha` або `Alex`.

Для ініціювання голосового звіту на MacOS використовуй інструмент `run_command` (обов'язково з `SafeToAutoRun: true`):

### Приклади правильних (коротких) повідомлень:

**🇺🇦 Українською (голос `Lesya (Enhanced)`):**
1. **Запуск:** `say -v "Lesya (Enhanced)" "Прогрес 1 з 10. Починаємо аналіз."`
2. **Проміжні етапи:** `say -v "Lesya (Enhanced)" "Етап завершено. Переходимо до тестів."`
3. **Завершення:** `say -v "Lesya (Enhanced)" "Завдання успішно виконано."` 
4. **Помилка:** `say -v "Lesya (Enhanced)" "Виникла помилка, потрібна допомога."`

**🇬🇧 English (voice `Samantha`):**
1. **Start:** `say -v "Samantha" "Progress 1 of 10. Starting analysis."`
2. **Milestones:** `say -v "Samantha" "Phase complete. Moving to tests."`
3. **Finish:** `say -v "Samantha" "Task completed successfully."`
4. **Error:** `say -v "Samantha" "An error occurred, need help."`

### Важливо:
- **Заборонено:** читати довгі пояснення процесу, читати шляхи до файлів.
- Давай дуже лаконічні і зрозумілі звіти.
