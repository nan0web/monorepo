# Універсальний Шаблон Проєкту: @nan0web/ai

> **Призначення:** Цей пакет слугує єдиним стандартизованим інтерфейсом (ядром) для взаємодії з мовними моделями (LLM) у всій екосистемі `nan.web`, абстрагуючи провайдерів та реалізуючи стратегії "розумного" вибору моделей.

---

## 🧭 Фаза 1: Філософія та Абстракція (The Seed)

1. **Місія та Філософія:**
   - Пакет створює Абсолютну Незалежність (Sovereignty) від конкретних AI-провайдерів (OpenAI, Cerebras, HuggingFace тощо).
   - мИ отримуємо можливість змінювати "розум" системи (LLM) на льоту, не переписуючи бізнес-логіку додатків. Це прояв волі до суверенітету коду.
2. **Абстракція (Базовий Додаток):**
   - Headless ядро. Пакет не має прив'язки до інтерфейсу. Це чиста абстракція потоків (`streamText`, `generateText`), що відповідає за отримання знань та звітність по ресурсах (`Usage`).
3. **Термінологія (Глосарій):**
   - `ModelInfo` — метадані моделі (контекст, ціна, потужність).
   - `AiStrategy` — розумний вибір моделі за осями (`finance`, `speed`, `volume`, `level`).
   - `Usage` — облік ресурсів (токенів).
   - `TestAI` — детермінована модель-заглушка для тестів без мутації реального мІру (API).

---

## 📐 Фаза 2: Доменне Моделювання (Data-Driven Models)

1. **Створення Model.js:**
   - Моделі (`ModelInfo.js`, `Usage.js`, `Pricing.js`) описані як чисті структури даних. Вони є основою для будь-якої взаємодії.
   - Метадані моделей використовуються для автоматичного підбору (`AiStrategy`), а не просто для візуалізації.
2. **Metadata Types (`@type`):**
   - Поки що пакет оперує чистими JS абстракціями без UI-анотацій (`@type`), оскільки він є фундаментальним ядром для інших пакетів екосистеми.
3. **Data Sources (Джерела Даних як Основа):**
   - Відсутні власні бази даних. Дані — це конфігурації провайдерів та змінних середовища. Провайдери реєструються на льоту (`ModelProvider`).
4. **Vector Indexing (Sovereign Cache):**
   - Індексація використовує `VectorDB` та кешування блоків (`MarkdownIndexer`). Зберігаються хеш-суми для кожного чанку.
   - Це дозволяє переіндексовувати лише змінені блоки, уникаючи даремних витрат на LLM Embeddings.

---

## 🛠 Фаза 3: Верифікація Логіки (CLI-First)

1. **Інтеграція з `ui-cli`:**
   - Ця фаза в самому пакеті реалізована через чисті функції та 100% покриття unit-тестами з використанням `node --test` та `TestAI`. CLI інтерфейси реалізуються споживачами (напр., через `ui-cli`).
   - `TestAI` гарантує, що логіка додатків, які залежать від цього пакета, може бути протестована ізольовано і без витрат.

---

## 🪐 Фаза 4: Sovereign Workbench (The Master IDE)

_Ця фаза наразі не реалізована в пакеті._  
Оскільки `@nan0web/ai` — це абстрактне ядро, власної пісочниці UI він наразі не має. Його абстракції та моделі використовуються в Master IDE інших пакетів та додатків екосистеми `nan.web` (наприклад, для генерації коду чи контенту).

---

## 🎨 Фаза 5: Тематизація та Інтерфейс (Theming)

_Ця фаза наразі не реалізована в пакеті._  
Пакет `@nan0web/ai` не містить жодного UI/CSS. Відокремлення логіки від бренду тут абсолютне: пакет є чистою логікою, цілком позбавленою візуального відображення. Брендування чи рендеринг реалізуються виключно пакетами-споживачами.
