# 🏗️ Architecture: @nan0web/catalog-watch

> Lightweight catalog change subscription via HTTP HEAD + ETag.
> Zero-State Server principle: the server generates static files, the client polls.

---

## 🧭 Phase 1: Philosophy & Abstraction (The Seed)

### 1.1 Mission

Give any OLMUI application (web, CLI, PWA, AI agent) the ability to **detect catalog changes without a backend** — using only HTTP HEAD and SHA-256 hashes in a plain-text index file. Zero WebSockets, zero push notifications, zero server-side runtime logic.

### 1.2 Abstraction

The package implements the **Pull-based Catalog Subscription** pattern:

```
[SSG Build] → .index.txt (plain text, SHA-256) → CDN/Static
                                                     ↓
[Client] → HTTP HEAD → ETag/Hash Compare → Updated? → ask(download) → mutate state
```

This is **not** a real-time system. It is **poll-based** with minimal traffic: one `HEAD` request (0 bytes body) per hour.

### 1.3 Glossary

| Term | Definition |
|------|-----------|
| **Catalog** | A named collection of products (cards, metals, credits, news) |
| **Index** | A plain-text `.index.txt` file with metadata and file listing |
| **Watcher** | A client subscriber polling the Index via HTTP HEAD |
| **Hash** | SHA-256 (first 12 chars) of the concatenated file contents |
| **ETag** | HTTP header automatically generated by CDN for static files |
| **Intent** | OLMUI yield object (`progress`, `log`, `ask`, `result`) |

## 📐 Phase 2: Domain Modeling (Data-Driven Models)

### 2.1 CatalogIndexModel (`src/domain/CatalogIndexModel.js`)

**Purpose:** Server-side index file model. Generates and parses `.index.txt`.

**Schema (static fields):**

| Field | Type | Default | Purpose |
|-------|------|---------|---------|
| `catalog` | string | `''` | Catalog name |
| `locale` | string | `'en'` | Language code |
| `version` | number | `0` | Auto-incremented build version |
| `hash` | string | `''` | SHA-256 (12 chars) |
| `itemCount` | number | `0` | Number of items |
| `updatedAt` | string | `''` | ISO 8601 timestamp |

**Serialization:**
- `toString()` → plain text with `#` headers + one file per line
- `CatalogIndexModel.parse(text)` → reconstruction from text

**Generator:** `async *run(env)` — scans directory, hashes contents, increments version, yields result.

### 2.2 CatalogWatcherModel (`src/domain/CatalogWatcherModel.js`)

**Purpose:** Client-side subscription model. Platform-agnostic — works everywhere.

**Schema (static fields):**

| Field | Type | Hint | Default | Purpose |
|-------|------|------|---------|---------|
| `url` | string | url | `''` | Index URL |
| `interval` | number | slider | `3600` | Interval (sec) |
| `lastHash` | string | hidden | `''` | Previous hash |
| `lastCheck` | string | hidden | `''` | Last timestamp |
| `status` | string | badge | `'idle'` | State (5 values) |
| `autoConfirm` | boolean | toggle | `false` | Auto-confirm |

**States (Status enum):** `idle` → `checking` → `updated` | `unchanged` | `error`

**Generators:**
- `async *check(env)` — single check: HEAD → ETag → GET → parse → compare → ask → mutate
- `async *watch(env)` — infinite loop: `check()` → `sleep(interval)` → repeat

### 2.3 Adapters

Three adapters bridge agnostic models to real environments:

- **CatalogWatcher** (EventBus — Browser/Node.js): `.on('updated')`, `.start()`, `.stop()`
- **buildCatalogIndex** (SSG Plugin — Node.js): `node:fs` + `node:crypto`, outputs `.index.txt`
- **Service Worker** (`sw.js` — PWA): Cache API, `visibilitychange` trigger

### 2.4 i18n

27 translation keys extracted from `static UI`. Base (EN) + Ukrainian (UK):
- `data/_/t.yaml` — English
- `data/uk/_/t.yaml` — Ukrainian
