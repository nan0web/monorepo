# Завдання: Прототипування ядра `@nan0web/learn`

## 🎯 Ціль
Розробити базовий алгоритмічний контракт (Model-as-Schema) для пакета `@nan0web/learn`, який математично описує процес навчання (Стипендійну Економіку). Цей пакет є математичною основою для вивчення будь-якої дисципліни (мови, програмування тощо).

## 🛠 Кроки виконання на майбутнє

1. **Ініціалізація пакета:**
   - Створити структуру пакета `@nan0web/learn`.
   - Встановити залежності (посилання на `@nan0web/db`, `@nan0web/types`).
   - Налаштувати `package.json` та конвеєр перевірки TypeScript (JSDoc).

2. **Опис моделі `LearnerStateModel`:**
   - Створити клас `LearnerStateModel.js` (як джерело правди про прогрес).
   - Поля: `userId`, `currentDelta` (Словник/Знання, що очікують вивчення), `motivationIndex` (Числа).
   - Поле `isHumanVerified` (boolean) — флаг проходження 2LW (Two Live Witnesses).

3. **Створення ядра економіки (Scholarship Model):**
   - Розробити `ScholarshipModel.js` для розрахунку вартості підписки на основі балів.
   - Метод `calculateNextPeriodDiscount(earnedPoints)`:
     - 100+ балів = 100% знижка.
     - 50 балів = 50% знижка.
     - 0 балів = 0% знижка.

   - Метод `calculateBonus(earnedPoints, isHumanVerified)`: нарахування унікальних крипто/кешбек бонусів і стипендій *ТІЛЬКИ* для верифікованих людей (офлайн-реєстрація, захист від ботів).

4. **Розробка Контрактних Тестів (TDD):**
   - Написати серію тестів (напр. з використанням `node:test`).
   - *Кейс 1:* Користувач вчиться ідеально і отримує 100% знижку на наступний тиждень.
   - *Кейс 2:* Боту (без 2LW-верифікації) відмовляють у стипендії або надлишку бонусів.
   - *Кейс 3:* Розрахунок `Delta * Motivation` для видачі наступної порції знань.

5. **Інтеграція в OLMUI Console:**
   - Розробити команди для UI-CLI: `learn study` (старт сесії), `learn stats` (показ балів та майбутньої знижки).
