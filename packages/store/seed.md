# 🛒 Намір: NaN0Web Store (Global App Store & Registry)

## 📌 Суть (The Seed)
`@nan0web/store` — це центральне сховище та механізм дистрибуції для додатків (`apps/`), пакетів (`packages/`), ШІ-намірів та плагінів від сторонніх розробників. У своєй найпершій версії (MVP) він слугує жорсткою та надійною базою даних метаданих для `SearchSourcesIntent`.

## 🚀 MVP (Мінімально Життєздатна Версія)
Нам потрібно максимально швидко запуститися, щоб індекс та AI-пошук запрацювали. 

1. **Структура БД (DB-FS)**
   Усі дані про зареєстровані пакети зберігаються у класичній файловій структурі. Щоб уникнути обмежень файлових систем (наприклад, коли в одній директорії понад 1 мільйон додатків, як в App Store, `readdir` створює жорсткі затримки), ми застосовуємо **Sharding (Шардинг)** за першими двома літерами `org_name`:
   ```text
   data/
     packages/
       @n/a/                   # Перші літери: @n, a
         @nan0web/
           ai/
             manifest.yaml
       @j/o/
         @john-doe/
           super-plugin/
   ```
   *Завдяки такому шардингу (грідингу), кожна директорія міститиме не більше кількох сотень папок, що забезпечить блискавичне завантаження і масштабування до мільйонів додатків.*
   **Обов'язкові поля `manifest.yaml`**: `name`, `description`, `version`, `tags` (масив), `repository`. Версіонування є **критичним** — ми маємо чітко знати, яка версія додатка заіндексована і пропонується користувачу.

2. **Store Builder (Агрегатор)**
   Окрема логіка (у межах Store), яка сканує всі папки всередині `data/packages/` і перетворює їх на один плоский еталонний файл `nan0web_store.csv` у кореню. Завдяки доданню поля `version`, `csv` тепер міститиме: `name, workspace, path, tags, version, description`. Це гарантує, що AI не буде галюцинувати щодо старих API.

## 🗺 Подальший розвиток (Наступні фази, не для MVP)
- **Відгуки (Reviews System):** Користувачі зможуть залишати відгуки через GitHub Discussions або напряму через сервер `@nan0web/db-server`. Відгуки зберігатимуться поруч із `manifest.yaml`.
- **Install Manager:** Команди `nan0cli store search <ім'я>` та `nan0cli store install <ім'я>` для модульного скачування і розгортання в Workspace.
