Тому що:

- `HTML` — це відображення (для браузера, пошти, UI)
- Цей пакет — **міст між логікою і формою**

> _"тИ не створює HTML.

## 📦 Як це працює?

### 1. `HTMLTransformer` — перетворює `nano` → `HTML`

- Приймає об’єкт або масив
- Повертає відформатований HTML
- Автоматично:
  - Додає теги (`div`, `p`)
  - Обробляє `.class`, `#id`
  - Обгортає `ul`/`ol` у `li`
  - Екранує небезпечні символи
  - Додає `<!DOCTYPE>`, коментарі, `script`, `style`

### 2. `HTMLTags` — налаштування за замовчуванням

- `$default` — який тег використовувати за відсутності
- `$nonEmptyTags` — які теги не можуть бути самозакривними
- `$tagAttrs` — відображення `. → class`, `# → id`
- `ul`, `ol`, `table` — основні теги для списків і таблиць

### 3. `escape(input)` — запобігання XSS

```js
escape('<script>alert("xss")</script>')
// → "&lt;script&gt;alert(&quot;xss&quot;)&lt;/script&gt;"
```

> ✅ Безпечність — не опція. Це — **стан**.

## 🌱 Приклад: Створення сторінки

```js
import { HTMLTransformer } from '@nan0web/html'

const tr = new HTMLTransformer()

const page = [
  { '!DOCTYPE': true, $html: true },
  {
    lang: 'uk',
    html: [
      { head: [{ meta: true, $charset: 'UTF-8' }, { title: 'мІр — порядок через дію' }] },
      {
        body: [
          { h1: 'Привіт, мІр' },
          { ul: ['Перший крок', 'Другий крок', { li: { strong: 'Реальна дія' } }] },
        ],
      },
    ],
  },
]

const html = await tr.encode(page)
console.log(html)
```

**Результат:**

```html
<!DOCTYPE html>
<html lang="uk">
  <head>
    <meta charset="UTF-8" />
    <title>мІр — порядок через дію</title>
  </head>
  <body>
    <h1>Привіт, мІр</h1>
    <ul>
      <li>Перший крок</li>
      <li>Другий крок</li>
      <li><strong>Реальна дія</strong></li>
    </ul>
  </body>
</html>
```

> ✅ Цей приклад — тест.  
> ✅ Цей тест — довірений.
