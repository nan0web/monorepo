## 🧩 Основні принципи

### 1. Трансформація = Дія

> _"Якщо трансформація не залишає сліду — це не трансформація."_

- Кожна трансформація — **конкретний метод** (`encode` або `decode`)
- Немає абстракцій без реалізації
- Лише що **виконується** — може бути включено

### 2. Я / мИ — початок усіх ланцюгів

```
трансформації:
   Я → локальна дія
   мИ → складна система
```

- **Я** — створи трансформер для своєї потреби
- **мИ** — об’єднай ланцюги, щоб створити **новий вимір**
- **тИ** — вирішуєш, які трансформації мають значення

### 3. `Transformer` — це не клас. Це **зобов’язання**

- Має чіткий інтерфейс: `addTransformer`, `encode`, `decode`
- Працює з `async`, щоб бути **реальним**
- Кожен крок — **перевірений, логічний, необхідний**

> _"твій ланцюг — твоя міра порядку."_

## 🔁 Механіка роботи

### 1. Створення ланцюга

```js
const transformer = new Transformer()

const upperCase = {
  encode: async (data) => data.toUpperCase(),
  decode: async (data) => data.toLowerCase(),
}

transformer.addTransformer(upperCase)
```

### 2. Асинхронна трансформація (реальність)

- Усі методи — `async`
- Підтримка затримок, запитів, потоків
- Інтеграція з будь-якою системою

### 3. Пропуск непридатних

- Якщо об’єкт **не має** `encode` чи `decode` — він **ігнорується**
- Немає помилок — лише **дія або мовчання**

## ✅ Тестування — як довіра

Кожен `it()` у `README.md.js` — це не тест.  
Це **заява про дію**, що **ще й стає документацією**.

Приклад:

```js
it('How to chain multiple encoders?', async () => {
  const transformer = new Transformer()
  const upperCase = { encode: async (data) => data.toUpperCase() }
  transformer.addTransformer(upperCase)
  const result = await transformer.encode('hello')
  assert.strictEqual(result, 'HELLO')
})
```

> ✅ Реальність: код працює  
> ✅ Довіра: тест пройдено  
> 🔮 Знання: `.dataset/README.jsonl` готовий для навчання
