## 1. Концепція

`ui-voice` — голосовий адаптер в архітектурі OLMUI ("One Logic, Many UI").
Це п'ятий інтерфейс поряд із `ui-lit`, `ui-cli`, `ui-chat` та `ui-api`.

```
ui-lit   → App.run(msg) → DOM / Web Components
ui-cli   → App.run(msg) → Terminal stdout
ui-chat  → App.run(msg) → Chat message
ui-api   → App.run(msg) → HTTP JSON response
ui-voice → App.run(msg) → Speech audio output
```

Голосовий інтерфейс складається з двох компонентів:

- **STT (Speech-to-Text)** — розпізнавання голосу користувача → текст → `App.run(msg)`
- **TTS (Text-to-Speech)** — відповідь системи → голос

### 5.1. Інтерфейс адаптера

```js
/**
 * Voice adapter for the OLMUI ecosystem.
 *
 * @example
 * import { VoiceAdapter } from '@nan0web/ui-voice'
 *
 * const voice = new VoiceAdapter({
 *   stt: 'browser',     // 'browser' | 'whisper-local' | 'whisper-api'
 *   tts: 'browser',     // 'browser' | 'piper' | 'openai'
 *   lang: 'uk-UA',
 * })
 *
 * // Listen → transcribe → run app → speak
 * voice.onTranscript = async (text) => {
 *   for await (const output of app.run({ body: { text } })) {
 *     await voice.speak(output.content)
 *   }
 * }
 *
 * voice.listen()
 */
export class VoiceAdapter {
  constructor(options) {
    /* ... */
  }
  async listen() {
    /* Start recording & recognition  */
  }
  async speak(text) {
    /* Generate & play audio response  */
  }
  stop() {
    /* Stop listening                  */
  }
}
```

## 8. Response Formatter (Двошаровий TTS)

TTS не повинен читати таблиці, JSON, JSX-код чи YAML.
Кожна відповідь `ui-api` повертає об'єкт з двома шарами:

```js
{
  // Шар 1: Для голосу (TTS) — лаконічний висновок дії
  speech: "Додано темну варіацію Хедера. Мова — англійська.",

  // Шар 2: Для чату (UI) — повний контекст з вкладеннями
  chat: {
    text: "Створено нову варіацію `shell-header` з параметрами:",
    attachments: [
      { type: 'code', language: 'jsx', content: '<Header theme="dark" locale="en" />' },
      { type: 'diff', content: '+ theme: dark\n+ locale: en' },
      { type: 'file', name: 'header-dark.yaml', url: '/sandbox/exports/header-dark.yaml' }
    ]
  }
}
```

### Правила форматування для TTS:

1. **Тільки висновок дії** — що зроблено, скільки елементів змінено.
2. **Без коду, таблиць, шляхів** — це йде в `chat.attachments`.
3. **Файли — як вкладення** для завантаження/перегляду, якщо пристрій підтримує.
4. **Максимум 2 речення** для `speech`.

## 9. BYOK: Bring Your Own Key (Суверенна Модель Доступу)

### Концепція

Кожен користувач самостійно вводить свій API-ключ для обраного провайдера.
Платформа **не зберігає ключі на сервері** — тільки в `localStorage` клієнта.

```yaml
# Налаштування в localStorage або профілі користувача
voice:
  stt: deepgram # web-speech | deepgram | whisper-api | whisper-local
  stt_key: dg_abc123... # BYOK: ключ користувача (null для безкоштовних)
  tts: edge-tts # web-speech | edge-tts | elevenlabs | openai-tts | piper
  tts_key: null # edge-tts безкоштовний, ключ не потрібен
  llm: cerebras # cerebras | groq | openai | gemini | deepseek
  llm_key: sk-xyz789... # BYOK: ключ користувача
  llm_model: llama-3.3-70b # модель за замовчуванням
```

### Переваги BYOK:

1. **Нульовий операційний кост** — ми не платимо за API-запити користувачів.
2. **Суверенність** — користувач сам обирає провайдера та контролює витрати.
3. **Без vendor lock-in** — зміна провайдера = зміна одного рядка у налаштуваннях.
4. **Приватність** — дані голосу не проходять через наші сервери.
5. **Масштабованість** — при 10,000 користувачів ми все ще платимо $0 за їхні запити.

### Порівняння LLM-провайдерів для BYOK

| Провайдер                    | Ціна (input / output за 1M токенів) | Швидкість      | Нотатка                    |
| ---------------------------- | ----------------------------------- | -------------- | -------------------------- |
| **Cerebras** (Llama 3.3 70B) | ~$0.60 / $0.60                      | **Найшвидший** | Вже інтегрований в nan.web |
| **Groq** (Llama 3.3 70B)     | $0.59 / $0.79                       | Дуже швидкий   | Гарний fallback            |
