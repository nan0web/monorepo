## 🧭 Фаза 1: Філософія та Абстракція (The Seed)

### Місія

`@nan0web/xml` — довірений атом для двостороннього перетворення між nan•стилем JS-об'єктів та XML. Жодного DOM, жодних зовнішніх залежностей, жодного зайвого байта.

### Філософія

- **Мінімалізм — це довіра.** Кожна функція вичерпна і має тест.
- **Тест → Документація → Знання.** Кожен `it()` у `README.md.js` є одночасно тестом, прикладом і запитом для LLM.
- **Код без дії — шум.** Якщо тест не проходить — пакет не живе.

### Абстракція

Пакет реалізує контракт `Transformer` з `@nan0web/transformer` (базовий клас з методами `encode(data)` / `decode(str)`), розширюючи його XML-специфікою.

### Термінологія

| Термін        | Визначення                                                     |
| ------------- | -------------------------------------------------------------- |
| nan•об'єкт    | Lightweight JS-об'єкт з конвенціями `$attr`, `#comment`, `?pi` |
| `$attr`       | Атрибут XML-тега (ключ починається з `$`)                      |
| `#comment`    | XML-коментар `<!-- text -->`                                   |
| `?xml`        | Processing Instruction `<?xml ... ?>`                          |
| `defaultTags` | Конфігурація маппінгу тегів, self-closing, CDATA               |
| `$selfClosed` | Логіка самозакриття тегу (`<br />`)                            |
| `$cdataTags`  | Теги, чий контент огортається у `<![CDATA[...]]>`              |
| `$tagAttrs`   | Маппінг CSS-селекторних нотацій на атрибути (`div.main#id`)    |

## 📐 Фаза 2: Доменне Моделювання (Data-Driven Models)

### Архітектура модулів

| Модуль                           | Роль                                                              |
| -------------------------------- | ----------------------------------------------------------------- |
| `Case`                           | Перетворення рядків між camel, kebab, snake, pascal, upper, lower |
| `escape(unsafe, ignore)`         | Екранування XML-спецсимволів (`&`, `<`, `>`, `"`, `'`)            |
| `nano2attrs(attrs, defaultTags)` | Серіалізація `$attr` об'єкта → XML-рядок атрибутів                |
| `nano2xml(data, options)`        | Конвертація nan•об'єкта/масиву → XML-рядок                        |
| `XMLTags`                        | Конфігурація маппінгу тегів та self-closing логіки                |
| `XMLTransformer`                 | OOP-обгортка: `encode()` + `decode()` (extends `Transformer`)     |

### Джерела даних

- Конфігурація тегів через `XMLTags` (in-code, екземпляри класу)
- RSS/Atom фіди — основний use-case для `decode()` (парсинг зовнішніх XML)

## 🗺 Дорожня Карта

### v1.1.0 — `decode()` (XML → nano)

- Імплементація `XMLTransformer.decode()` — зворотне перетворення XML → nan•об'єкт
- RegExp-based парсер без зовнішніх залежностей
- Підтримка RSS/Atom фідів (`eaukraine.eu` pipeline)
- Round-trip верифікація: `encode(data) → xml → decode(xml) → data₂` ≈ `data`
- Документовано у `REQUESTS.md`

### Майбутнє

- `@nan0web/rss` — пакет для роботи з RSS-фідами (залежить від `decode()`)
- Streaming XML parser для великих файлів
- XSLT-подібні трансформації через nan•шаблони
