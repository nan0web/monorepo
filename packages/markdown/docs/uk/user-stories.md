# Користувацькі історії: Аудит цілісності документації

## Історія 1: Перевірка внутрішніх посилань
**Як** Архітектор,
**Я хочу**, щоб аудитор підтверджував, що всі відносні посилання в папці `docs/` вказують на існуючі файли,
**Щоб** користувачі не стикалися з помилками 404 під час навігації.

### Сценарій: Посилання на існуючий файл
- **Дано**: файл `docs/README.md`, що містить `[Перейти до індексу](./index.md)`
- **І**: файл `docs/index.md` існує
- **Коли** я запускаю `nan0inspect provendocs .`
- **Тоді** повідомлень про помилки для цього посилання не повинно бути.

### Сценарій: Посилання на відсутній файл
- **Дано**: файл `docs/README.md`, що містить `[Бите посилання](./missing.md)`
- **І**: `docs/missing.md` НЕ існує
- **Коли** я запускаю `nan0inspect provendocs .`
- **Тоді** я повинен побачити помилку: `Broken link in docs/README.md: ./missing.md`

## Історія 2: Перевірка якорів (анкорів)
**Як** Архітектор,
**Я хочу** бути впевненим, що всі якорі (заголовки), на які посилаються в лінках, дійсно існують у цільових файлах,
**Щоб** глибокі посилання працювали коректно.

### Сценарій: Валідний локальний якір
- **Дано**: `docs/README.md` має заголовок `## Можливості`
- **І**: посилання `[Можливості](#mozhlivosti)`
- **Коли** я запускаю аудит
- **Тоді** помилок якорів не повинно бути.

### Сценарій: Відсутній локальний якір
- **Дано**: `docs/README.md` НЕ має заголовка `## Налаштування`
- **І**: посилання `[Налаштування](#nalashtuvannya)`
- **Коли** я запускаю аудит
- **Тоді** я повинен побачити помилку: `Missing anchor #nalashtuvannya in docs/README.md`

## Історія 3: Структурна цілісність
**Як** Архітектор,
**Я хочу**, щоб аудитор вимагав обов'язкову структуру документації (index, README, langs),
**Щоб** проект залишався зручним для пошуку та перекладу.

### Сценарій: Відсутній індекс
- **Дано**: `docs/index.md` відсутній
- **Коли** я запускаю аудит
- **Тоді** я повинен побачити помилку про відсутність індексу.
