# 🎨 @nan0web/ui-react-bootstrap

> **Призначення:** Це основна бібліотека візуальних компонентів для складних веб-додатків екосистеми (таких як Industrial Bank). Вона перетворює Доменні Моделі у живі, редаговані та брендовані інтерфейси.

---

## 🧭 Фаза 1: Філософія та Абстракція (The Seed)

1. **Місія:** Надати інструментарій для створення "Живих Моделей" (Living Models), де UI є лише відображенням даних.
2. **Абстракція:** Пакет не містить бізнес-логіки, лише адаптери між `@nan0web/core` та DOM.
3. **Глосарій:**
   - **Welder (Зварювальник):** Система автоматичного підключення (welding) даних до компонентів через Interface Welding Contract.
   - **Variation:** Стан компонента, збережений як набір CSS-класів та пропсів.

---

## 📐 Фаза 2: Доменне Моделювання (Data-Driven Models)

1. **Model-as-Schema:** Компоненти форм автоматично споживають `ProjectModel` та інші класи, генеруючи ввідні поля на основі `static` метаданих.
2. **Типи:** Підтримка складних типів, таких як `text/markdown` (через Prosemirror) та рекурсивні `Model[]`.

---

## 🛠 Фаза 3: Верифікація Логіки (CLI-First)

1. **Snapshots:** Кожен React компонент покривається HTML-снепшотами для перевірки рендерингу в різних станах.

---

## 🪐 Фаза 4: Sovereign Workbench (The Master IDE)

1. **OlmuiInspector:** Вбудована пісочниця для кожного компонента.
2. **Variation Manager:** Можливість зберігати та перемикати стани компонентів прямо в браузері.

---

## 🎨 Фаза 5: Тематизація та Інтерфейси (Theming)

1. **Bootstrap 5:** Використання стандартних сіток та утиліт.
2. **Custom Tokens:** Керування брендами через SCSS змінні.
3. **ComponentSandbox:** Візуальний редактор стилів, інтегрований у бібліотеку.

---

## ✅ Фаза 6: Якість та Дистрибуція (Quality & Distribution)

1. **ProvenDocs:** README.md.js, який демонструє кожен компонент у живих тестах.
2. **Release:** Автоматична публікація в npm.

---

## 💎 Фаза 7: Цінність та Економіка (Value & Economy)

1. **Цінність:** Скорочення часу на редизайн цілої екосистеми додатків через зміну одного токена в ядрі.

---

## 🌐 Фаза 8: Децентралізація та Суверенітет (Sovereignty)

1. **Self-contained:** Компоненти не потребують зовнішніх CDN або API для базового рендерингу.

---

## 🌍 Фаза 9: Місія та Вплив (World Impact)

1. **Мета:** Зробити якісний фінансовий дизайн доступним для швидкої розробки суверенних додатків.
