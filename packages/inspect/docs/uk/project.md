# 🕵️ @nan0web/inspect

- [🇺🇦 Українська (Оригінал)](./docs/uk/project.md)
- [🇬🇧 English (Translation)](./docs/en/project.md)

**Призначення:** Пакет `@nan0web/inspect` є "очима" системи, що дозволяє іншим агентам розуміти структуру та якість будь-якого пакету в монорепозиторії, дотримуючись суворих стандартів Zero-Hallucination.

---

## 🧭 Фаза 1: Філософія та Абстракція (The Seed)

- **Місія:** Забезпечити абсолютну цілісність інженерії (Zero-Hallucination) шляхом автоматизованої інспекції екосистеми.
- **Відповідальність:** Валідація Model-as-Schema стандартів, перевірка циклічних залежностей та надійності структури проекту.

## 📐 Фаза 2: Доменне Моделювання (Data-Driven Models)

Доменна логіка аудиторів (Domain):
- `ArchitectureAuditor`: Головний координатор з перевірки загальної інженерної якості за протоколами `project.md`.
- `CircularDependencyAuditor`: Шукає цикли в графах модулів ESM.
- `NoTypeScriptAuditor`: Гарантує відсутність нативного `.ts` у `src/`.
- `StructureAuditor`: Перевіряє файлову структуру (наявність `data/`, `play/`, `README.md.js`).

_(Зверніть увагу: `StackDetector` перенесено з цього пакету у `protocol`, для підтримки архітектурної чистоти)._

## 🛠 Фаза 3: Верифікація Логіки (CLI-First)

Пакет не містить самостійного термінального бінарника, проте його моделі інтегровані в команди екосистемного CLI інспектора для миттєвої валідації.

## 🪐 Фаза 4: Sovereign Workbench (The Master IDE)

*(Відсутня для цього пакету: Sandbox-пісочниця не вимагається для автономних аудиторів, що діють в рамках тестів екосистеми.)*

## 🎨 Фаза 5: Тематизація та Інтерфейси (Theming)

*(Відсутня для цього пакету: усі результати мапляться на `ui` моделі типу повідомлень OLMUI).*
