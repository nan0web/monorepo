# 🌍 Ворклоу: i18n Builder

Цей workflow є **Третім Кроком** у конвеєрі оркестрації (після `@[/model-schema]`). Його мета — створити або оновити ієрархічні YAML-словники на базі згенерованої моделі.

## 1. Атрибутика Контексту

- **Вимога до чату:** 🔴 NEW CHAT (Обнулений контекст).
- **Необхідні файли на вхід:**
  - Згенерований файл моделі: `src/{Name}Model.js`.
  - Стандарти даних: `@[/data-architecture]` та `@[/i18n-standards]`.
  - Поточні мови: `data/_/langs.yaml`.
  - Поточні переклади: `data/_/t.yaml`.

## 2. Завдання Агента (i18n Generator / Model-First i18n)

Ти МАЄШ:

1. Проаналізувати `static` поля з переданого `{Name}Model.js`. Витягнути всі тексти (напр. з `help`, `validate`, або хардкод-тексти з помилок).
2. Зчитати `data/_/langs.yaml`, щоб дізнатися, скільки локалей підтримує цей пакет (наприклад, `en`, `uk`). Якщо такого файлу немає, створити `en` за замовчуванням.
3. Сформувати ієрархічні ключі для `_/t.yaml` відповідно до принципів `Model-First i18n`. Ключі мають точно повторювати шлях до властивості моделі (напр., `Models.{Name}Model.city`).
4. Оновити базовий `.yaml` та всі локалізовані версії (наприклад, `data/uk/_/t.yaml`).
5. **Заборонено!** Використовувати модулі Node.js `fs` у сзгенерованому коді пакетів для парсингу YAML. Тільки `@nan0web/db-fs` або аналог для читання. Під час самого генерування YAML використовуй інструмент `write_to_file`.

## 3. Результат Кроку

- Створені або оновлені файли YAML у відповідних папках.
- Повідомлення користувачу:
  > "✅ Переклади згенеровано. Перейди до наступного кроку `@[/forge-component]` у НОВОМУ ЧАТІ, зібравши потрібний контекст."
